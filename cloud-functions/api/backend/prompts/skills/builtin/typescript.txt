- Enable TypeScript strict mode (strict: true in tsconfig). No exceptions.
- Never use `any` — use `unknown` and narrow with type guards. For truly dynamic data, use `zod` schemas for runtime validation.
- Prefer discriminated unions over optional properties for modeling state variants. Each variant should be a distinct type.
- Use `satisfies` operator for exhaustiveness checks on union types.
- Leverage template literal types and brand types (e.g., `type UserId = string & { __brand: 'UserId' }`) for domain primitives.
- Function signatures must declare return types explicitly for all exported functions. Use `Result<T, E>` pattern instead of throwing exceptions.
- Prefer `readonly` arrays and objects by default. Use `as const` assertions for literal inference.
- Use `interface` for public API contracts (extensibility), `type` for unions/intersections/mapped types.
- Avoid type assertions (`as`) except in test setup or when narrowing from `unknown`. Prefer type guards.
- Generics: constrain with meaningful bounds. Use descriptive type parameter names (`TItem` not `T`).
